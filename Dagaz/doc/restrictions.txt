- Общие ограничения ZoG
  - Размещение фигур
!   - Клетка доски не может содержать более двух фигур
      - Ур (Скирюка)
      - Таврели
      - Столбовые шашки
      - Ритмомахия (пирамиды)
      - "Гонки" на доске Сёги (взятая сверху фигура "подвозит" взявшую её на своем ходе)
      - Transport Chess
    - Отсутствует возможность определения "тождественных" полей (точки склейки)
      - Star Trek 3D Chess - "точки склейки" перемещаются при движении малых досок
    - Отсутвует возможность определения зон, связанных с фигурами
      - Телохранитель (Хиашатар)
      - Василиск (D&D Chess)
      - Шут (Omega Chess)
  - Генерация ходов
    - Фазы хода
!     - Инициализация (отсутствует)
      - Ход (отсутствует многократное выполнение для частичных ходов)
!     - Проверки выполнения инвариантов (отсутствует)
        - Шахматы - не нахождение Короля под шахом после своего хода (нарушение - мат)
          - Поскольку инвариант наложен на своего Короля, чужих шаховать можно
        - Шашки - выбор хода с максимальным взятием (нарушаемый инвариант)
          - Ранее выбранные ходы (с меньшим взятием) отвергаются при уточнении инварианта
          - Новые ходы уточняют инвариант (изменяя переменную, по которой выполняется проверка)
?       - Сёги - нельзя ставить мат сбросом пешки
?       - Шатар - сложная система матования (нельзя ставить мат Конем)
        - Forks - снятие фигур, попадающих под вилку (в том числе с многократным выполнением)
        - Шашки - превращение в дамку при остановке на поле превращения (побочный эффект проверки инварианта)
      - Проверка условий завершения
        - repetition
        - Отсутствует возможность отмены проверки 3-кратного повторения позиции
    - "Змея" (Космические шахматы) - ход диагональным зигзагом в пределах 3 ортогоналей
      - Проблема определения достижимости клеток на центральной ортогонали
        - Достижима если можно пройти через одну из 2 клеток на боковых ортогоналях
    - Нельзя сгенерировать ход выполняющий только взятия (в ZSG такая возможность имеется)
    - Отсутствие поддержки relative-config при генерации хода
      - Мельница - при построении 3 в ряд (только при завершении хода), снимается произвольная фигура противника
      - Чатуранга - триумф Слонов
    - add - постановка фигуры + завершение формирования хода (исправлено в Axiom)
!   - Можно ходить только своими фигурами
      - Голубой ферзь - нейтральная фигура которой может ходить любой из противников
        - В ZoG ходы нейтральными фигурами должны быть определены в turn-order явно
        - Для реализации в ZoG требуется сделать нейтральными все фигуры
      - Ставропольские шашки
      - Енохианские шахматы - игра чужими фигурами после захвата трона
    - Взятие фигур, вышедших за пределы поля
      - Abalone
      - Ур
  - Передача хода (отсутствуют команды явной передачи хода)
    - Ур - пропуск хода противником при завершении хода на Розетке (приходится выполнять Pass)
    - Fanorona (реализована при помощи частичного хода)
?   - Seega
  - Коалиции (могут изменяться в процессе игры)
    - Енохианские шахматы - дружественные короли не угрожают друг другу
  - AI
    - Плохая поддержка игр с количеством игроков не равным 2
    - Плохая поддержка игр в которых игроки делают по несколько ходов
    - Плохая поддержка частичных ходов
    - Отсутствие поддержки игр с неполной информацией
      - Стратего - фигуры не скрываются от AI
      - Ур - не учитываются вероятности выпадения очков при рассчете на несколько ходов
?     - Ур - возможность предварительной генерации ПСЧ для рассчета на неограниченное количество ходов вперед
      - Battle vs Chess - сокрытие части правил от AI
    - Отсутствует возможность подключения дебютных и эндшпильных библиотек, а также библиотек эвристик
?   - Отсутсвует возможность отсечения повторяющихся позиций (использовать Zobrist hashing)
      - При рассчете текущего хода
      - Учет позиций по состоянию на предыдущие ходы
        - Го - Ко борьба
    - Отсутствие возможности рандомизации принятия решения
  - Интерфейс
    - Поддержка сложных траекторий перемещения фигур
    - Поддержка шкал, отображающих глобальные значения (в том числе, изменяемые в реальном времени)
    - Поддержка одновременного перемещения фигур, входящих в перемещаемую группу
      - Рыжий осёл - плашки сложной формы состоят из нескольких фигур
    - Поддержка трансляции записи ходов из ZSG в общепринятый (для данной игры)
      - В том числе при выборе хода из нескольких возможных вариантов
    - Возможность отключения режима "перетаскивания" фигур
      - Рыжий осёл
- Ограничения ZSG
  - Не поддерживаются операции с группами фигур в одной клетке
  - Не фиксируются изменения глобальных значений (Axiom) и атрибутов фигур
    - Если глобальное значение изменяется случайным образом, партия не может быть воспроизведена по ZSG
  - Не поддерживается добавление фигур в резерв
  - Не поддерживается перемещение досок
    - Star Trek 3D Chess
    - TURN-THE-TABLE CHECKERS [http://skyruk.livejournal.com/334328.html] (фрагменты доски могут поворачиваться по кругу)
- Ограничения ZRF
  - Генерация ходов
!   - Не допускается формирование нескольких add-partial в цикле
      - Шашки - при бое, Дамка может остановиться не только на следующей клетке
    - Pass - явный пропуск хода (реализовано в Axiom)
    - Отсутствие арифметики
      - Ритмомахия
    - Фигура совершающая ход, рассматривается как находящаяся но доске, в исходной точке (исправлено в Axiom)
    - change-owner не работает в играх для более чем 2 игроков 
?   - Отсутствие возможности переопределения команд
      - Переопределение capture для добавления в резерв
  - Условия завершения
    - checkmated
      - Неявный контроль инварианта + условие завершения
      - Не работает при более чем 2 игроках (инвариант контролируется только для предыдущего игрока по turn-order)
      - Bomb Chess - не дает съесть фигуру, которая может подорвать бомбы, в зоне действия которых находится Король
    - Отсутсивие возможности кастомизации проверки условий завершения
      - Ритмомахия (построение гармоний)
      - Квадраты (подсчет площади)
?     - Pay Sho
      - Thud (подсчет стоимости фигур)
      - Го (рассчет захваченной территории)
- Ограничения Axiom
  - Размещение фигур
    - Поддерживаются только единственный 2-мерный grid
  - Генерация ходов
    - Не поддерживаются атрибуты фигур
      - Шахматы (проверка того, что фигура не двигалась до рокировки)
      - hitpoint-ы фигур
    - Не поддерживаются зоны
  - AI
?   - Непонятно как генерировать продолжения частичных ходов для их оценки
  - Взаимодействие с ZoG
    - Отсутсвует возможность передачи начальной расстановки фигур из ZoG
- Реализованные возможности ZoG
  - Размещение фигур
    - Возможность определения запрещенных позиций
      - Отсутствует возможность запрета остановки при разрешенном прохождении
!     - Отсутствует возможность определения "штрафных" полей (удаление фигуры при задержке на поле более чем на N ходов)
    - Определение симметрий
  - Генерация ходов
    - attacked? (проверка нахождения поля под боем)
      - Проверка поддержки поля своей фигурой
      - Числовое значение (количество фигур, атакующих поле)
    - Приоритеты
      - Ставропольские шашки
        - Не поддерживаются приоритеты в зависимости от игрока
        - Не поддерживается назначение одного хода многим приоритетам (или конструкция ИЛИ в списке приоритетов)
    - Частичные ходы
    - mark/back
      - Не поддерживаются вложенные сохранения
    - Проверка результата перехода по направлению (реализовано в Axiom)
      - Для работы с направлениями необходимо изменять here (текущее расположение)
    - Сбросы в частичных ходах (реализовано в Axiom)
    - Использование стека (реализовано в Axiom)
!     - Splut! - реализация хода Гнома
  - Передача хода
    - repeat
  - Условия завершения
    - relative-config
      - Hasami Shogi - победа при построении 5 в ряд
    - absolute-config
?     - Asalto - захват всех клеток зоны
- AI
    - Iterative deepening with transposition table
    - Limited move reordering based on 'best move from previous iteration' stored in the transposition table
    - Full width searching
  - Интерфейс
    - Smart Move
      - Нажатие на фигуру, если для неё возможен только один ход
        - В том числе для взятий и переключений
      - Нажатие на поле если в него возможен только один ход
        - В том числе для сбросов
?     - Ход в единственном направлении не возвращающем в предыдущее состояние
